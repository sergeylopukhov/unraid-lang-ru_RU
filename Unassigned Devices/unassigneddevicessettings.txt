:ud_settings_plus_plug:
> **Примечание:** для монтирования HFS+ и exFAT, а также для включения режима разрушительных операций должен быть установлен плагин **Unassigned Devices Plus**.
:end

:ud_settings_common_script_plug:
> Общий скрипт запускается при событиях udev и hotplug для устройств Unassigned Devices.
:end

:ud_settings_destructive_plug:
> Если включено, Unassigned Devices сможет стирать и форматировать разделы. Параметр нельзя включить без плагина **Unassigned Devices Plus**.
:end

:ud_settings_mount_usb_devices_plug:
> Если выбрано **Yes**, новое USB-устройство будет автоматически смонтировано и опубликовано без участия пользователя.
:end

:ud_settings_discard_plug:
> Если выбрано **Yes**, SSD будут монтироваться с параметром discard, когда файловая система это поддерживает.
:end

:ud_settings_symlinks_plug:
> Установите **Yes**, чтобы создать symlink для /mnt/remotes/ в /mnt/disks/.
>
> Это нужно для обратной совместимости привязок VM и Docker Container к точке монтирования /mnt/disks/.
> Рекомендуется изменить привязки на точку монтирования /mnt/remotes/.
>
> Если обратная совместимость не нужна, оставьте **No**.
:end

:ud_settings_remote_share_wait_plug:
> Время ожидания в секундах перед монтированием удалённых общих ресурсов при первом запуске массива.
:end

:ud_settings_specify_smb_vers_plug:
> Установите **No**, чтобы Unassigned Devices сначала пробовал монтировать удалённые SMB-ресурсы без указания версии SMB.
>
> Установите **Yes**, чтобы Unassigned Devices монтировал удалённые SMB-ресурсы только с явно заданной версией SMB.
>
> Для некоторых удалённых серверов версию нужно указывать явно.
:end

:ud_settings_debug_level_plug:
> Debug Log Level добавляет подробное журналирование в выбранных областях UD для диагностики.
>
> **None** отключает отладочные журналы. **Udev and Hotplug** включает диагностику udev, обнаружения дисков и hotplug. **Refresh and Update** помогает искать зависания UD. **Command times** показывает время выполнения команд и пишет очень много сообщений.
>
> Не оставляйте Debug Log Level включённым надолго: он может быстро заполнить syslog.
:end

:ud_settings_specify_nfs_vers_plug:
> **Auto** монтирует все удалённые NFS-ресурсы с автоматическим выбором версии NFS.
>
> Если **Auto** не работает или монтирует только NFSv3, используйте **NFSv4**, потому что ваша версия Unraid может не согласовывать NFSv4 автоматически.
:end

:ud_settings_smb_security_plug:
> Уровень SMB Sharing:
>
> + **Disabled** отключает общий доступ SMB.
> + **Public** публикует устройства, Remote SMB/NFS-ресурсы и ISO mounts с доступом на чтение и запись.
> + **Private** включает SMB Security и позволяет задавать доступ по пользователям.
> + **Private Hidden** включает SMB Security для скрытых SMB-ресурсов.
>
> Windows Server Signing: если SMB-ресурсы не открываются в Windows 11 24H2 или новее, создайте в Unraid пользователя с паролем и тем же именем, что у учётной записи Windows, либо настройте Windows credentials вручную.
:end

:ud_settings_case_sensitive_plug:
> Включает или отключает чувствительность к регистру для имён, где это поддерживается.
:end

:ud_time_machine_plug:
> Установите **Yes**, чтобы включить настройки Time Machine для SMB-ресурсов.
:end

:ud_time_machine_vol_size_plug:
> Ограничивает отображаемый размер тома, чтобы Time Machine не использовала всё реальное место на диске для резервной копии. Например, значение "1024" ограничит отображаемый размер до 1 ГБ.
:end

:ud_settings_force_user_plug:
> Иногда права на SMB-ресурсах обрабатываются не так, как ожидается. Эта функция может помочь с такими правами.
>
> Она применяется ко всем устройствам, смонтированным через Unassigned Devices. Недостаток: переопределяются групповые права Linux. Если не уверены, оставьте выключенной.
:end

:ud_settings_nfs_security_plug:
> Установите **Yes**, чтобы экспортировать Unassigned Devices через NFS. Общие ресурсы экспортируются как Public с чтением и записью или как Private с заданным правилом. Доступ управляется правами файлов и папок.
:end

:ud_settings_nfs_rules_plug:
> **Примечание:** обычное правило UD: '*(rw,sec=sys,insecure,anongid=100,anonuid=99,no_root_squash)'. Меняйте правила NFS только если понимаете требования клиентов и последствия для доступа.
:end

:ud_settings_disk_pass_plug:
> Укажите пароль зашифрованного диска. Оставьте пустым, чтобы использовать пароль/парольную фразу массива.
:end

:ud_settings_change_uuid_plug:
> Выберите диск, UUID которого нужно изменить. Это полезно, когда XFS-диск удалён из массива и имеет тот же UUID, что диск массива.
>
> Диск должен быть размонтирован. **Примечание:** не меняйте UUID диска, входящего в пул.
:end

:ud_settings_zfs_upgrade_plug:
> Выберите диск, который нужно обновить до последней версии ZFS. Для обновления ZFS Pool диск должен быть смонтирован.
:end

:ud_remote_server_test_plug:
> На удалённых серверах должны быть включены SMB и/или NFS. Проверка убеждается, что соответствующие порты открыты, а команда 'showmount -e <server>' в Unraid показывает доступные NFS mounts.
>
> Для статуса SMB online у удалённого сервера должен быть открыт SMB-порт. Для NFS online должен быть открыт NFS-порт и 'showmount -e' должен показывать NFS-ресурсы.
>
> Если порт открыт, но 'showmount -e' не показывает NFS shares, NFS работать не будет. Для Synology и похожих серверов это обычно означает ошибку экспорта или прав доступа.
:end
