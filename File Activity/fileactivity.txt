:file_activity_logging_plug:
> Мониторинг файловой активности:
>
> Открытие, чтение, запись и изменение файлов отслеживаются через inotify, пишутся в журнал массива и отображаются по дискам, общим ресурсам, UD-дискам и кэшу.
> Чтобы вести журнал активности дисков, нужно запустить File Activity.
> File Activity рассчитан на короткие периоды работы, чтобы проверить активность дисков.
> На сервере с высокой файловой активностью место под журналы может быстро заполниться.
> Каталоги `appdata`, `docker`, `syslogs` и `system` исключены без учёта регистра.
>
> Примечание: мониторинг File Activity останавливается при остановке массива и снова запускается при старте массива, если он включён.
:end

:file_activity_enable_logging_plug:
> Установите **Yes**, чтобы включать мониторинг File Activity при запуске сервера.
:end

:file_activity_enable_ud_plug:
> Установите **Yes**, чтобы включить мониторинг File Activity для Unassigned Devices, если установлен соответствующий плагин.
:end

:file_activity_enable_cache_plug:
> Установите **Yes**, чтобы включить мониторинг File Activity для Cache и Pool Disks. Pool disks применимы только в Unraid 6.9.
:end

:file_activity_display_events_plug:
> Количество файловых событий из журнала File Activity, показываемых на дисках и общих ресурсах для каждого ресурса или диска.
:end
