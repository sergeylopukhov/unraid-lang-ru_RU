:tips_tweaks_general_plug:
> Общие рекомендации:
>
> + Статический IP-адрес иногда помогает, особенно для NFS-ресурсов, которые привязаны к IP. Зарезервируйте адрес на роутере или используйте высокий адрес, чтобы избежать конфликтов.
> + Jumbo Frames обычно не рекомендуются: они могут вызывать проблемы с некоторыми Docker-контейнерами.
> + Не задавайте vm.dirty_ratio выше 20%, если нет UPS, который надёжно выключит Unraid. При сбое питания можно потерять данные, ещё не записанные на диск.
> + vm.dirty_background_ratio выше половины vm.dirty_ratio не рекомендуется, кроме случаев, когда оба значения однозначные.
:end

:tips_tweaks_nic_flow_control_plug:
> Значение 'Yes' отключает NIC Flow Control. Эта функция появилась для сетей со смешанными скоростями и сейчас часто не нужна.
>
> На управляемом коммутаторе Flow Control обычно можно включить или отключить на стороне коммутатора. На бытовом коммутаторе такой настройки может не быть. Если есть сетевые проблемы, рекомендуется отключить Flow Control.
>
> Переход с любого значения на 'Default' требует перезагрузки.
:end

:tips_tweaks_nic_offload_plug:
> Значение 'Yes' отключает NIC offloads. Offload переносит часть сетевой обработки на NIC и снижает нагрузку на CPU, но TSO и другие offload-функции могут вызывать сетевые проблемы.
>
> Если есть сетевые сбои, рекомендуется отключить NIC Offloads. Переход с любого значения на 'Default' требует перезагрузки.
:end

:tips_tweaks_rx_buffer_plug:
> Размер кольцевого буфера NIC Rx. У некоторых NIC есть ограничение на размер буфера. Рекомендуемое значение по умолчанию - 256.
>
> Переход с любого значения на 'Default' требует перезагрузки.
:end

:tips_tweaks_tx_buffer_plug:
> Размер кольцевого буфера NIC Tx. У некоторых NIC есть ограничение на размер буфера. Рекомендуемое значение по умолчанию - 256.
>
> Переход с любого значения на 'Default' требует перезагрузки.
:end

:tips_tweaks_nics_plug:
> Перечислите NIC через запятую, чтобы применить к ним указанные выше настройки. Если несколько NIC объединены в bond, обязательно укажите все. Настройки Flow Control, TSO и буферов NIC будут
> применены к этим NIC.
:end

:tips_tweaks_dirty_background_plug:
> Для ВМ потокового видео и игр задавайте низкие однозначные значения. Рекомендуемые значения: vm.dirty_background_ratio=2% и vm.dirty_ratio=3%.
> При большом объёме RAM меньшие значения обычно лучше.
> Внимание: не увеличивайте эти значения выше настроек по умолчанию, если нет UPS, который надёжно выключит Unraid.
>
> 'vm.dirty_background_ratio' задаётся как процент от общего объёма RAM. Обычно рекомендуется 10%; для media/gaming ВМ - 2%, при OOM - 1%.
:end

:tips_tweaks_dirty_ratio_plug:
> 'vm.dirty_ratio' задаётся как процент от общего объёма RAM.
> Слишком высокое значение может заставить Linux завершать процессы ради освобождения памяти. Обычно рекомендуется 20%; для media/gaming ВМ - 3%, при OOM - 2%.
:end

:tips_tweaks_writeback_plug:
> Значение по умолчанию - 200 сантисекунд (2 секунды). Оно задаёт, как часто ядро проверяет dirty data для фоновой записи на диск.
:end

:tips_tweaks_expire_plug:
> Значение по умолчанию - 1000 сантисекунд (10 секунд). Оно задаёт, как долго данные могут оставаться dirty, прежде чем будут рассматриваться для write-back.
:end

:tips_tweaks_cpu_governor_plug:
> Выберите обычный CPU Scaling Governor.
>
> + "Power Save" - меняет частоту CPU по нагрузке.
> + "On Demand" - стандартный для ACPI CPU Freq, агрессивно меняет частоту по нагрузке.
> + "Conservative" - мягче меняет частоту, но может добавить задержку.
> + "Performance" - держит CPU на полной частоте независимо от нагрузки; может улучшить производительность и задержки.
> + "Schedutil" - использует планировщик Linux для выбора частоты CPU.
>
> Governor можно задать командой: /usr/local/sbin/rc.tweaks set_governor 'governor'. Доступны 'ondemand', 'performance', 'powersave', 'conservative', 'schedutil'.
:end

:tips_tweaks_power_save_cpu_governor_plug:
> Выберите CPU Scaling Governor для режима энергосбережения.
>
> + "Power Save" - меняет частоту CPU по нагрузке.
> + "On Demand" - стандартный для ACPI CPU Freq, агрессивно меняет частоту.
> + "Conservative" - мягче меняет частоту, но может добавить задержку.
> + "Schedutil" - использует планировщик Linux для выбора частоты CPU.
:end
